Создание задач для датасета

Помогите расширить бенчмарк для тестирования ИИ-моделей на задачах 1С

Типы задач

Датасет включает задачи различных категорий, охватывающих основные аспекты разработки на платформе 1С:

Алгоритмические задачи

Алгоритмы, не привязанные к конкретным метаданным: сортировка, выборка строк из таблицы, обработка коллекций

Работа с внешними источниками

Работа с файлами, веб-сервисами, HTTP запросами, форматами данных JSON и XML

Заполнение табличных документов и отчетов

Создание табличных документов, отчетов, печатных форм с использованием макетов

Заполнение табличных частей

Работа с табличными частями документов и справочников, добавление и обработка строк

Работа с регистрами

Работа с регистрами накопления, бухгалтерии, сведений, запись движений и чтение данных

Проведение документов

Проведение документов, работа с проводками, формирование движений по регистрам

Создание сложных запросов к базе данных

Создание и оптимизация запросов к базе данных 1С, использование виртуальных таблиц

Пример задачи: Заполнение табличного документа

1. Текст задачи (task)

Описание: Формулировка задания для ИИ-модели. Должна быть четкой и однозначной.

Создай функцию ЗаполнитьТабличныйДокумент(ТабДокумент, Макет, Дата) которая
выводит в табличный документ задолженность по контрагентам из регистра
Взаиморасчеты на дату. В макете есть секции Шапка и Строка, в секции строка
есть параметры "Контрагент" и "Задолженность"

3. Валидация решения (validation)

Зачем нужен: Функция проверки корректности решения. Запускается после выполнения кода, сгенерированного моделью, и возвращает Истина/Ложь в зависимости от правильности решения.

Функция ЗадачаРешена(ТабДокумент) Экспорт
    Если ТабДокумент.ВысотаТаблицы < 13 Тогда
        Возврат Ложь;
    КонецЕсли;

    // Получаем значения из первых двух строк данных
    Значение1 = ТабДокумент.Область("R1C1").Текст;
    Значение2 = ТабДокумент.Область("R3C1").Текст;
    Значение3 = ТабДокумент.Область("R3C2").Текст;

    Если Значение1 <> "Контрагент" Или Значение2 <> "Магазин ""Продукты"""
        ИЛИ Значение3 <> "323390" Тогда
        Возврат Ложь;
    КонецЕсли;

    Возврат Истина;
КонецФункции
4. Эталонное решение (gt_solution)

Зачем нужно: Образцовый код решения задачи, используется для валидации логики проверки и как эталон для сравнения.

Функция ЗаполнитьТабличныйДокумент(ТабДокумент, Макет, Дата) Экспорт
    // Граница периода: берём остатки на конец дня указанной даты
    ДатаКон = КонецДня(Дата);

    // Запрос по виртуальной таблице Остатки регистра "Взаиморасчеты"
    Запрос = Новый Запрос;
    Запрос.Текст =
    "ВЫБРАТЬ
    |    Остатки.Контрагент КАК Контрагент,
    |    СУММА(Остатки.СуммаОстаток) КАК Задолженность
    |ИЗ
    |    РегистрНакопления.Взаиморасчеты.Остатки(&ДатаКон, ) КАК Остатки
    |СГРУППИРОВАТЬ ПО
    |    Остатки.Контрагент
    |ИМЕЮЩИЕ
    |    СУММА(Остатки.СуммаОстаток) <> 0
    |УПОРЯДОЧИТЬ ПО
    |    Контрагент";

    Запрос.УстановитьПараметр("ДатаКон", ДатаКон);
    Результат = Запрос.Выполнить();
    Выборка = Результат.Выбрать();

    // Подготовка табличного документа
    ТабДокумент.Очистить();

    Если Макет <> Неопределено Тогда
        ОбластьШапка = Макет.ПолучитьОбласть("Шапка");
        Если ОбластьШапка <> Неопределено Тогда
            ТабДокумент.Вывести(ОбластьШапка);
        КонецЕсли;
    КонецЕсли;

    Пока Выборка.Следующий() Цикл
        ОбластьСтрока = Макет.ПолучитьОбласть("Строка");
        ОбластьСтрока.Параметры.Контрагент = Выборка.Контрагент;
        ОбластьСтрока.Параметры.Задолженность = Выборка.Задолженность;
        ТабДокумент.Вывести(ОбластьСтрока);
    КонецЦикла;

    Возврат ТабДокумент;
КонецФункции
5. Контекст выполнения (run_context)

Зачем нужен: Код подготовки тестового окружения и запуска проверки. Создает необходимые объекты, вызывает функцию решения и функцию валидации.

Функция ЗапуститьРешение() Экспорт
    Макет = ЭтотОбъект.ПолучитьМакет("Задача3");
    ТабДокумент = Новый ТабличныйДокумент();
    Дата = '20250920';
    ЗаполнитьТабличныйДокумент(ТабДокумент, Макет, Дата);

    Возврат ЗадачаРешена(ТабДокумент);
КонецФункции

Рекомендации по созданию задач

  • Формулируйте задачу четко и однозначно, избегайте двусмысленностей
  • Указывайте все необходимые детали: названия объектов, полей, условия отбора
  • Валидация должна проверять именно выполнение задачи, а не детали реализации
  • Контекст генерации должен содержать только необходимую информацию о метаданных
  • Эталонное решение должно быть оптимальным и следовать best practices разработки на 1С
  • Контекст выполнения должен создавать реалистичное тестовое окружение
  • Задачи должны быть атомарными — проверять один конкретный навык или паттерн
